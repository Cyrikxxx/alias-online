## 📖 О проекте

**Alias Online** — полноценная многопользовательская игра реального времени
с серверной игровой логикой, двумя режимами, защитой от подглядывания и
продуманной отказоустойчивостью.

Проект построен как **монорепозиторий из трёх пакетов** и развёрнут на четырёх
облачных сервисах, каждый из которых отвечает за свою задачу. Фронтенд и
WebSocket-сервер делят **общий типизированный контракт** — поэтому клиент и
сервер физически не могут «разъехаться» по форматам событий.

> 🎯 Два режима в одной игре: **онлайн** (каждый со своего телефона) и
> **локально** (одно устройство передаётся по кругу) — без регистрации,
> вход в комнату по 6-символьному коду.

## ✨ Возможности

### 🎮 Для игроков

- 🌐 **Онлайн-комнаты** — хост создаёт комнату, остальные заходят по коду с любого устройства
- 📱 **Локальный режим** — одно устройство на компанию: передавайте телефон по кругу, без второго гаджета и без комнат
- ⏱️ **Живой таймер раунда** — отсчёт ведёт сервер, у всех участников идёт синхронно
- 🧩 **10 категорий, 600+ слов** — от животных до абстрактных понятий
- 👥 **2–6 команд**, гибкая настройка времени раунда и очков для победы
- ✅ **Сверка итогов раунда** — объясняющий поправляет спорные слова перед передачей хода
- ⏸️ **Пауза, переподключение, история игр** — ничего не теряется при обрыве связи
- 🌗 **Светлая и тёмная темы**, адаптив под телефон и десктоп

### 🛠️ Под капотом

- 🔄 **Серверная машина состояний** — фазы раунда считает сервер, клиент только рисует
- 🔐 **Приватная доставка слова** — слово видит только объясняющий, через DevTools не подсмотреть
- 🪪 **HMAC-токены вместо cookie** — авторизация работает между разными доменами (Vercel ↔ Railway)
- ⚡ **Redis для «горячего» состояния + Postgres для истории** — каждое хранилище по назначению
- 🧬 **End-to-end типобезопасность** — общий пакет типов для фронта и сокет-сервера
- 🚀 **Полностью автоматический CI/CD** — `git push` → пересборка обоих сервисов

## 🏗️ Архитектура

```
                          🌐 Браузер игрока
                         /                  \
              HTTPS (страницы,           WSS (реалтайм:
               REST /api/*)               лобби, раунды)
                       │                       │
                       ▼                       ▼
            ┌────────────────────┐   ┌────────────────────┐
            │  Vercel            │   │  Railway           │
            │  apps/web          │   │  apps/ws           │
            │  Next.js + REST    │   │  Socket.io сервер  │
            └─────┬────────┬─────┘   └─────┬────────┬─────┘
            Prisma│        │ioredis  Prisma│        │ioredis
                  ▼        ▼               ▼        ▼
            ┌──────────────┐         ┌──────────────────────┐
            │  Neon        │         │  Upstash Redis       │
            │  PostgreSQL  │         │  снапшоты комнат      │
            │  (история)   │         │  (TTL 24ч)           │
            └──────────────┘         └──────────────────────┘
```

| Сервис | Роль | Почему именно так |
|---|---|---|
| **Vercel** | Страницы Next.js + REST API (`/api/*`) | Идеален для serverless-фронта и быстрых API |
| **Railway** | Постоянный WebSocket-сервер | Vercel-функции умирают после ответа — держать живое WS-соединение там нельзя |
| **Neon** | PostgreSQL — пользователи, игры, словарь, история | Source of truth, переживает перезагрузки |
| **Upstash** | Redis — состояние онлайн-комнаты | Быстрый «горячий» кеш: фаза, таймер, очередь слов |

> 💡 **Ключевое решение:** Vercel и Railway не общаются напрямую. Связь идёт через
> общий снапшот комнаты в Redis и через подписанный токен. Это позволяет
> масштабировать каждую часть независимо.

## 🔐 Как устроена защита реалтайма

Cookie с домена `*.vercel.app` физически не доедет до `*.railway.app`.
Поэтому авторизация WebSocket построена на **подписанном токене**:

```
1. Браузер создаёт/входит в комнату  ──REST──▶  Vercel
2. Vercel подписывает токен HMAC-секретом (userId, roomCode, role, exp)
3. Браузер открывает WebSocket       ──WSS───▶  Railway, передаёт токен
4. Railway проверяет подпись ТЕМ ЖЕ секретом  ──▶  пускает в комнату
```

А **слово для объяснения** отправляется адресно — только сокету объясняющего.
Остальные игроки получают лишь карточку «X объясняет» и счётчик очков.
Открыть DevTools и подсмотреть слово соперника **невозможно** — его там просто нет.

## 🗄️ Модель данных

Postgres хранит долговременное состояние через **Prisma ORM** (13 моделей):

| Группа | Модели |
|---|---|
| 📚 Словарь | `Category`, `Word`, `WordCategory` |
| 👤 Идентичность | `User` (анонимная, без пароля) |
| 🚪 Онлайн-комнаты | `Room`, `RoomCategory`, `Participant` |
| 🎲 Игры | `Game`, `GameCategory`, `Team`, `Player`, `Round`, `RoundWord` |

Онлайн-комната в момент игры живёт в Redis как снапшот (`room:<code>`) и
сбрасывается в Postgres на старте и в конце каждого раунда — поэтому партия
восстанавливается даже после перезапуска сервера.

## 🔧 Переменные окружения

Локально — в файле `.env` в корне (он в `.gitignore`, секреты в репозиторий не попадают).
На проде задаются в дашбордах Vercel и Railway.

| Переменная | Где нужна | Описание |
|---|---|---|
| `DATABASE_URL` | Vercel + Railway | Строка подключения к PostgreSQL (Neon) |
| `REDIS_URL` | Vercel + Railway | Redis (Upstash), формат `rediss://…` |
| `WS_TOKEN_SECRET` | Vercel + Railway | HMAC-секрет токенов. **Должен совпадать байт-в-байт!** |
| `NEXT_PUBLIC_WS_URL` | Vercel | Публичный URL WS-сервера на Railway |
| `WS_CORS_ORIGIN` | Railway | URL приложения на Vercel (белый список CORS) |

## ☁️ Деплой и CI/CD

Полностью автоматический. После `git push` в ветку `main`:

- 🟢 **Vercel** пересобирает `apps/web` (~2–3 мин)
- 🔵 **Railway** пересобирает `apps/ws` — только если менялись `apps/ws/`,
  `packages/shared/` или `prisma/` (Watch Paths, ~1 мин)

Ручных шагов нет. Откат — одной кнопкой в дашборде.
